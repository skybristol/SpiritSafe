# SpiritSafe

The Entity Profile cache and export repository for the Global Knowledge Commons.

SpiritSafe stores the generated artifacts that bridge Data Distillery Wikibase content into offline, version-controlled files that the gkc runtime can consume directly.

## Repository Scope

- profiles/: generated JSON Entity Profiles, written one file per profile QID.
- cache/entities/: per-entity Wikibase cache JSON used as the build substrate for profile export.
- cache/refresh/: refresh summaries from recentchanges-driven cache updates.
- queries/: shared SPARQL query workspace.
- .github/workflows/cache-from-wikibase.yml: manual workflow that refreshes cache/entities and exports JSON profiles into profiles/.
- .github/workflows/validate-profile.yml: baseline structural checks.
- .github/workflows/pr-automerge.yml: auto-merge enablement for non-draft PRs targeting main.

## Cache And Export Flow

SpiritSafe now operates as a two-stage artifact repository:

1. Refresh cache/entities from Data Distillery Wikibase recentchanges.
2. Build JSON Entity Profiles from the refreshed cache and write them into profiles/.

The current GitHub Actions entrypoint for this is the manual workflow named Cache From Wikibase.

## Generated Profile Artifacts

Profile exports are generated by gkc from cache/entities and written as flat JSON files:

- profiles/Q4.json
- profiles/Q39.json
- profiles/<QID>.json

These files are committed build artifacts, not hand-authored source files.

## PR Policy

Non-draft pull requests to main are configured for squash auto-merge once required checks pass.
